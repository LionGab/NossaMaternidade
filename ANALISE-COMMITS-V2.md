# ANALISE-COMMITS-V2

## Optimized Commit Analysis
An effective commit analysis helps us maintain code quality, trace changes efficiently, and understand the impact of each commit on the overall project.

## Better Structure
- Each commit should be concise and describe the changes made.
- Utilize a consistent format for commit messages:
  - **[Type]: [Subject]**
  - **[Body]** (optional)

## Visual Aids
- Use graphs and charts to visualize commit trends over time.
- Consider tools like GitHub Insights or external tools for analytics.

## Decision Matrix
| Criteria       | Commit A | Commit B | Decision  |
|----------------|----------|----------|-----------|
| Size           | Large    | Medium   | Commit A  |
| Impact         | High     | Low      | Commit A  |
| Complexity     | Low      | Medium   | Commit A  |

## Business Impact Analysis
- Assess how each commit aligns with business goals.
- Prioritize commits that enhance features or fix critical bugs.

## Technical Metrics
- Commit Frequency: Number of commits per week.
- Churn Rate: Lines of code added vs. removed.
- Review Time: Average time for a commit review.

## Useful Commands
- `git log --oneline` : View commit history in concise format.
- `git diff <commit1> <commit2>` : View changes between two commits.
- `git blame <file>` : Identify who made changes to each line.
